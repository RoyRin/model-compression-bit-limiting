### 2B. Just Ask (Request-Based Compression)

**Hypothesis**: Asking model to write succinctly might improve compression.

**Key Finding**: Counterintuitively, verbose solutions compress BETTER than succinct ones.
- Verbose: 2115 chars, 5.3% compression
- Succinct: 589 chars, 9.5% compression
- **Reason**: Verbose text uses predictable boilerplate; dense text uses unusual phrasing

### 2C. Iterative Binary Questions (SLM/LLM/Question-Asker Loop)

**Hypothesis**: An SLM can be guided to produce LLM-quality answers through iterative yes/no questions. Each question = 1 bit of information transferred from LLM to SLM.

#### Core Loop (`iterative_SLM_loop` in `lossy_compression/core/qa_compression.py`)
1. **SLM generates initial answer** (cheap model)
2. **Question-Asker (SLM/Sonnet) generates binary question** about the answer
3. **LLM (Opus) answers yes/no** based on its knowledge
4. **SLM improves answer** using accumulated Q&A guidance
5. **Repeat** until quality threshold or max iterations

#### Models (from `lossy_compression/__init__.py`)
| Role | Default Model |
|------|---------------|
| LLM (Oracle) | `claude-opus-4-1-20250805` |
| SLM (Answerer) | `claude-haiku-4-5-20251001` |
| Question Generator | `claude-sonnet-4-5-20250929` |

#### Key Parameters Across All Benchmarks

| Parameter | Default | Description |
|-----------|---------|-------------|
| `--max-questions` / `-q` | 25-30 | Max Q&A iterations (bits of information) |
| `--batch` | False | Generate questions in batches |
| `--batch-size` | 10 | Questions per batch |
| `--difficulty` | varies | easy/medium/hard/all/medium+hard |
| `--verbose` | False | Show full responses |
| `--parallel` | False | Parallel execution (HumanEval) |
| `--max-workers` | 4 | Parallel workers |

| Mode | Used For | Key Difference |
